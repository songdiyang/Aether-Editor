# 前后端分离架构重构计划

## 目标
将 `aether-win32` crate 重构为两层架构：
- **前端层 (`frontend/`)**：平台无关的UI逻辑，只调用内核层接口
- **内核层 (`kernel/`)**：封装所有Windows API，提供跨平台抽象

## 当前问题分析

`window.rs` (3565行) 严重耦合，包含：
1. **Win32窗口管理**：`CreateWindowExW`, `RegisterClassW`, `window_proc`, `GetMessageW`
2. **Direct2D渲染**：`CreateSolidColorBrush`, `FillRectangle`, `DrawText` 等
3. **剪贴板操作**：`OpenClipboard`, `GlobalAlloc`, `SetClipboardData` 等
4. **文件对话框**：`IFileOpenDialog`, `IFileSaveDialog` (在 `dialogs.rs`)
5. **业务逻辑**：`new_file`, `load_file`, `save_file`, `undo`, `redo`, `cut`, `copy`, `paste`

## 重构任务

### Task 1: 创建内核层目录结构
新建 `crates/aether-win32/src/kernel/` 目录：
- `mod.rs` - 内核模块入口，定义平台抽象 trait
- `platform.rs` - 平台能力 trait 定义（跨平台接口）
- `windows/` - Windows 具体实现
  - `window.rs` - Win32窗口管理
  - `render.rs` - Direct2D渲染封装
  - `clipboard.rs` - 剪贴板操作
  - `dialog.rs` - 文件对话框
  - `mod.rs` - Windows实现入口

### Task 2: 定义平台抽象 trait
在 `kernel/platform.rs` 中定义：

```rust
// 平台渲染接口
pub trait PlatformRenderer {
    fn clear(&mut self, color: &Color);
    fn fill_rect(&mut self, x: f32, y: f32, width: f32, height: f32, color: &Color);
    fn draw_text(&mut self, text: &str, x: f32, y: f32, width: f32, height: f32, color: &Color, font_size: f32);
    fn begin_frame(&mut self);
    fn end_frame(&mut self) -> Result<(), RenderError>;
}

// 平台窗口接口
pub trait PlatformWindow {
    fn create(title: &str, width: u32, height: u32) -> Result<Self, WindowError>;
    fn resize(&mut self, width: u32, height: u32);
    fn get_dpi_scale(&self) -> f32;
    fn set_cursor_pos(&mut self, x: i32, y: i32);
    fn show(&mut self);
    fn run_event_loop(&mut self, callbacks: &mut EventCallbacks);
}

// 平台剪贴板接口
pub trait PlatformClipboard {
    fn set_text(&mut self, text: &str) -> Result<(), ClipboardError>;
    fn get_text(&mut self) -> Result<Option<String>, ClipboardError>;
}

// 平台对话框接口
pub trait PlatformDialogs {
    fn open_file(&mut self, title: &str, filters: &[(&str, &str)]) -> Option<PathBuf>;
    fn save_file(&mut self, title: &str, default_name: &str) -> Option<PathBuf>;
    fn open_folder(&mut self, title: &str) -> Option<PathBuf>;
}

// 事件回调
pub struct EventCallbacks {
    pub on_mouse_down: Box<dyn FnMut(f32, f32)>,
    pub on_mouse_move: Box<dyn FnMut(f32, f32, bool)>,
    pub on_mouse_up: Box<dyn FnMut()>,
    pub on_key_down: Box<dyn FnMut(VirtualKey, bool, bool)>,
    pub on_char: Box<dyn FnMut(char)>,
    pub on_resize: Box<dyn FnMut(u32, u32)>,
    pub on_paint: Box<dyn FnMut()>,
    pub on_dpi_changed: Box<dyn FnMut(f32)>,
    pub on_destroy: Box<dyn FnMut()>,
}
```

### Task 3: 迁移剪贴板操作到内核层
将 `window.rs` 中的 `set_clipboard_text` 和 `get_clipboard_text` 迁移到 `kernel/windows/clipboard.rs`：
- 新建 `kernel/windows/clipboard.rs`，实现 `WindowsClipboard` struct
- `window.rs` 中改为调用 `self.platform.clipboard.set_text(...)`

### Task 4: 迁移文件对话框到内核层
将 `dialogs.rs` 迁移到 `kernel/windows/dialog.rs`：
- 新建 `kernel/windows/dialog.rs`，实现 `WindowsDialogs` struct
- `execute_command` 和 `window_proc` 中改为调用 `self.platform.dialogs.open_file(...)`

### Task 5: 迁移渲染到内核层
将 `window.rs` 中所有 Direct2D 渲染调用封装到 `kernel/windows/render.rs`：
- 新建 `WindowsRenderer` struct，实现 `PlatformRenderer` trait
- 封装 `CreateSolidColorBrush`, `FillRectangle`, `DrawText`, `begin_draw`, `end_draw`, `clear`
- `render_menu_bar`, `render_submenu`, `render_activity_bar` 等改为使用 `PlatformRenderer` 接口

### Task 6: 迁移窗口管理到内核层
将 `run()` 和 `window_proc` 迁移到 `kernel/windows/window.rs`：
- 新建 `WindowsWindow` struct，实现 `PlatformWindow` trait
- `window_proc` 中的事件处理转换为对 `EventCallbacks` 的调用
- `main.rs` 中改为创建 `WindowsWindow` 并注册回调

### Task 7: 重构 execute_command 移除 hwnd 依赖
当前 `execute_command(&mut self, cmd: CommandId, hwnd: HWND)` 依赖 HWND。
重构为：
```rust
pub fn execute_command(&mut self, cmd: CommandId) {
    // 不再接收 hwnd，通过 platform.dialogs 调用
}
```

### Task 8: 重构 EditorState 移除 HWND 字段
将 `EditorState` 中的 `hwnd: HWND` 替换为平台抽象：
```rust
pub struct EditorState {
    // 移除: pub hwnd: HWND
    // 添加: pub platform: Box<dyn Platform>,
    // ... 其他字段不变
}
```

### Task 9: 重构 window_proc 中的快捷键处理
当前 `window_proc` 中直接调用 `Dialogs::open_file_dialog(hwnd, ...)`。
改为通过回调机制让前端处理：
```rust
// 内核层
on_key_down: Box<dyn FnMut(VirtualKey, bool, bool)>,

// 前端层
callbacks.on_key_down = Box::new(|vk, ctrl, shift| {
    if ctrl && vk == VirtualKey::O {
        if let Some(path) = editor.platform.dialogs.open_file("打开文件", &[]) {
            editor.load_file(path);
        }
    }
});
```

### Task 10: 清理前端层的 Windows 导入
重构完成后，`window.rs` 中应只保留：
```rust
use crate::kernel::platform::{PlatformRenderer, PlatformWindow, PlatformClipboard, PlatformDialogs, EventCallbacks};
use crate::kernel::windows::{WindowsPlatform, WindowsRenderer, WindowsWindow, WindowsClipboard, WindowsDialogs};
```

所有 `windows::Win32::*` 导入应只在 `kernel/windows/*.rs` 中出现。

## 注意事项
- 保持 `aether-core` 和 `aether-render` 不受影响
- `aether-render` 已经是渲染抽象层，但 `aether-win32` 中直接调用了 D2D API，需要进一步封装
- 逐步重构，每次一个子系统，确保编译通过后再进行下一步
